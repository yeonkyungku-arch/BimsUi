피그마 메이크 프롬프트 (최종본)

앱 버전 업로드 및 배포 관리 기능을 포함한 운영 CMS 화면을 설계한다.
이 시스템은 B2B/B2G 환경으로 안정성, 이력 관리, 감사 추적이 매우 중요하다.

⸻

핵심 정책
	1.	버전(version)은 불변이며 덮어쓸 수 없다.
	2.	동일 버전의 수정은 revision(리비전)으로 관리한다.
	3.	이미 배포된 버전은 overwrite가 불가능하다.
	4.	리비전 등록 시 릴리즈 노트 작성은 필수이다.
	5.	릴리즈 노트는 수정/삭제할 수 없으며 로그로 영구 보관된다.
	6.	모든 변경은 작성자, 시간, 사유가 기록되어야 한다.
	7.	리비전은 기존 배포 이력에 영향을 주지 않는다.
	8.	기존 단말에는 자동 반영되지 않으며, 재배포를 통해서만 적용된다.
	9.	동일 파일 업로드는 차단한다.
	10.	동일 버전 + 다른 파일 업로드 시 overwrite 대신 리비전 등록으로 유도한다.

⸻

설계할 화면

1. 앱 업로드 화면

구성 요소:
	•	파일 업로드 영역
	•	version 입력 필드
	•	변경 유형 선택
	•	신규 버전
	•	기능 동일 패치
	•	변경 사유 입력 필드
	•	릴리즈 노트 입력 (필수)

릴리즈 노트 안내 문구:
“변경 사항 및 수정 사유를 상세히 입력하세요. 입력 내용은 수정할 수 없으며 로그로 기록됩니다.”

유효성:
	•	릴리즈 노트 미입력 시 업로드 불가

⸻

2. 업로드 충돌 처리 모달

Case 1. 동일 파일 업로드
	•	메시지: “이미 등록된 파일입니다.”
	•	버튼: 확인 / 기존 버전 보기

Case 2. 동일 version + 다른 파일 (배포 이력 없음)
	•	메시지: “이미 등록된 버전입니다. 새 리비전으로 등록하시겠습니까?”
	•	버튼: 취소 / 리비전 등록

Case 3. 동일 version + 다른 파일 (배포 이력 있음)
	•	메시지:
“이미 배포된 버전입니다. 덮어쓸 수 없습니다.
기능 변경이 없는 수정은 새 리비전으로 등록할 수 있습니다.”
	•	버튼: 취소 / 패치 업로드 / 새 버전 등록

⸻

3. 리비전 등록 확인 모달
	•	릴리즈 노트 입력 (필수)
	•	메시지:
“기능 변경이 없는 수정으로 새 리비전을 등록합니다.
기존 배포 이력은 유지되며 이후 배포부터 적용됩니다.
입력한 릴리즈 노트는 수정할 수 없으며 로그로 기록됩니다.”

버튼:
	•	취소
	•	등록

⸻

4. 버전 상세 화면

상단 정보:
	•	앱명
	•	version
	•	최신 revision
	•	상태 (READY / DEPLOYED 등)
	•	마지막 배포 정보

리비전 리스트 테이블:

컬럼:
	•	revision (예: r1, r2)
	•	상태 (ACTIVE, SUPERSEDED)
	•	등록일시
	•	등록자
	•	변경 유형 (NEW / PATCH)
	•	변경 사유
	•	릴리즈 노트 요약
	•	배포 여부

액션:
	•	패치 업로드
	•	새 버전 생성
	•	배포 생성
	•	감사 로그 보기

⸻

5. 릴리즈 노트 상세 보기

리비전 클릭 시 상세 패널 또는 모달

표시 항목:
	•	version
	•	revision
	•	작성자
	•	작성일시
	•	변경 유형
	•	변경 사유
	•	릴리즈 노트 전체 내용

하단 안내:
“릴리즈 노트는 수정할 수 없으며 로그로 보관됩니다.”

⸻

6. 배포 생성 화면

입력 요소:
	•	version 선택
	•	revision 선택 (기본값: 최신)
	•	배포 대상 선택 (전체 / 그룹 / 단말)
	•	배포 일정 설정

옵션:
	•	“기존 설치 단말 포함하여 재배포” 체크박스

설명:
	•	체크 해제: 신규 단말만 배포
	•	체크 선택: 기존 단말도 업데이트

⸻

7. 감사 로그 화면

버전 또는 리비전 기준 로그 조회

로그 항목:
	•	이벤트 타입 (업로드, 패치, 배포 등)
	•	version / revision
	•	작업자
	•	작업 시간
	•	변경 사유
	•	릴리즈 노트

특징:
	•	로그 수정/삭제 불가
	•	시간순 정렬

⸻

UX 가이드
	•	운영자 중심 UI로 명확하고 직관적인 문구 사용
	•	“덮어쓰기” 대신 “패치 업로드”, “리비전 등록” 용어 사용
	•	위험 작업은 반드시 확인 모달 제공
	•	상태값은 배지 형태로 시각적으로 구분
	•	이력 정보는 테이블 중심으로 구성

⸻

핵심 메시지
	•	“이미 등록된 버전입니다.”
	•	“덮어쓸 수 없습니다.”
	•	“새 리비전으로 등록하시겠습니까?”
	•	“릴리즈 노트는 수정할 수 없으며 로그로 기록됩니다.”
