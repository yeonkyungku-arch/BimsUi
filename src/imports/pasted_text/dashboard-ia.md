BIMS 대시보드 화면 IA (MVP)

1. 화면 구조 (Layout IA)

[Header / Title]

[KPI 영역]  ← 정류장 기준 (고정)

--------------------------------

[탐색 영역]
  ├─ 뷰 선택 (고객사 뷰 / 정류장 뷰)
  ├─ 지도 (정류장 중심)
  ├─ 리스트 (뷰 기준)
  └─ 상세 패널


⸻

2. KPI 영역

목적
	•	전체 운영 상태 요약
	•	필터 트리거 역할

구성
	•	전체 정류장 수
	•	정상 정류장 수
	•	일부 장애 정류장 수
	•	전체 장애 정류장 수

⸻

인터랙션
	•	KPI 클릭 시
→ 정류장 기준 필터 적용
→ 지도 + 리스트 + 상세 패널에 반영

👉 뷰는 변경되지 않음

⸻

3. 뷰 선택 영역

위치
	•	탐색 영역 상단

구성
	•	고객사 뷰
	•	정류장 뷰

⸻

역할
	•	데이터 “구조”를 결정
	•	사용자 컨텍스트 유지

⸻

4. 지도 영역

기준
	•	정류장 중심

구성 요소
	•	정류장 핀
	•	정류장명
	•	대표 상태 (색상)
	•	소유 고객사
	•	공동 운영 여부

⸻

상태 표현
	•	대표 상태 = 정류장 내 단말 중 가장 위험한 상태

⸻

인터랙션
	•	정류장 클릭 시
→ 상세 패널 변경
→ 리스트 선택 동기화

⸻

5. 리스트 영역

5.1 고객사 뷰

구조

고객사 A
  - 정류장 1 (정상)
  - 정류장 2 (장애)

고객사 B
  - 정류장 3 (정상)


⸻

구성 요소
	•	고객사명
	•	연결 정류장 목록
	•	정류장명
	•	상태
	•	정류장 수
	•	이슈 정류장 수
	•	공유 정류장 여부

⸻

특징
	•	고객사 기준 그룹핑
	•	관리/운영 관점

⸻

5.2 정류장 뷰

구조

정류장 1 (정상)
정류장 2 (장애)
정류장 3 (정상)


⸻

구성 요소
	•	정류장명
	•	소유 고객사
	•	공동 운영자 여부
	•	단말 수
	•	대표 상태

⸻

특징
	•	플랫 리스트
	•	모니터링/운영 관점

⸻

6. KPI + 리스트 결합 로직

핵심 구조

👉 정류장 필터 → 리스트 구조 적용

⸻

동작 방식
	•	KPI 클릭
→ 정류장 필터 적용
	•	고객사 뷰
→ 필터된 정류장 기준으로 그룹핑
	•	정류장 뷰
→ 필터된 정류장 그대로 표시

⸻

결과
	•	고객사 구조 유지
	•	데이터만 변경
	•	조건에 맞지 않는 고객사 제거

⸻

7. 상세 패널

7.1 고객사 선택 시
	•	고객사명
	•	연결 정류장 목록
	•	이슈 정류장 수
	•	단말 수

⸻

7.2 정류장 선택 시
	•	정류장명
	•	소유 고객사
	•	공동 운영자
	•	단말 리스트
	•	단말명
	•	상태
	•	통신
	•	배터리
	•	OTA

⸻

8. 영역 간 인터랙션

액션	반응
KPI 클릭	필터 적용 (뷰 유지)
뷰 변경	리스트 구조 변경
지도 클릭	상세 + 리스트 선택 동기화
리스트 선택	상세 패널 변경


⸻

9. 핵심 UX 원칙
	•	KPI = 정류장 기준 필터
	•	뷰 = 사용자 선택 (고정)
	•	리스트 = 결과
	•	지도 = 위치 기반 탐색
	•	단말 = 실제 관리 대상
	•	고객사 = 그룹/책임 단위

⸻

10. 한 줄 요약

👉 “정류장 기준 KPI로 필터링하고,
선택한 뷰에 따라 리스트 구조를 유지한 채 결과를 보여주는 대시보드”
